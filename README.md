# testingpr

## Pull Request Templates

This repository includes multiple pull request templates to help you provide the right information for different types of changes.

### Available Templates

When creating a pull request, you can choose from the following templates:

1. **Bug Fix** (`bug_fix.md`) - Use this template when fixing a bug
   - Includes sections for description, root cause, changes, testing, and related issues

2. **Feature** (`feature.md`) - Use this template when adding a new feature
   - Includes sections for description, motivation, implementation details, testing, and demos

3. **Documentation** (`documentation.md`) - Use this template when updating documentation
   - Includes sections for type of documentation change, changes made, and review checklist

4. **Refactoring** (`refactor.md`) - Use this template when refactoring code
   - Includes sections for motivation, impact, and testing of refactored code

### How to Use

When creating a new pull request on GitHub:
1. Click "New pull request"
2. Add a title and description
3. GitHub will automatically prompt you to choose a template from the dropdown menu
4. Select the appropriate template for your change
5. Fill in the template sections

Alternatively, you can manually select a template by appending the template parameter to the PR URL:
- Bug fix: `?template=bug_fix.md`
- Feature: `?template=feature.md`
- Documentation: `?template=documentation.md`
- Refactoring: `?template=refactor.md`
